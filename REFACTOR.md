## 権限モデル（調査・実装済み）

### 管理者だけができること（APIレベルで403）

| 機能 | エンドポイント |
|------|------------|
| ユーザー管理 (CRUD) | `/users/*` |
| ロール管理 | `/roles/*` |
| システム設定 (LLM URL等) | `/settings/*` |
| API キー管理 | `/api-keys/*` |
| グループ作成・削除・メンバー操作 | `/groups/*` (一覧以外) |

### Unixスタイルパーミッション（ドキュメント・フォルダ毎）

```
rw  r-  r-
│   │   └── others (全員)
│   └────── group (所属グループ)
└────────── owner (常にrw固定)
```

評価の優先順位: `admin > owner > group > others`
- グループメンバーの場合、group の設定が適用され others にフォールバック**しない**
- フォルダの更新・削除は owner or admin のみ（実装済み）
- タグの作成・編集・削除は全ユーザー可（運用方針として決定）
